# Child Actor 101 Directory - Product Requirements Document (PRD)

**Version:** 3.0  
**Date:** February 14, 2026  
**Status:** Active Development / Production  
**Document Owner:** Development Team

---

## 1. Executive Summary

### 1.1 Product Vision
Child Actor 101 Directory is the definitive, high-velocity resource connecting families with verified entertainment industry professionals. The platform has evolved from a curated list into a **self-service, automated marketplace** where vendors can instantly manage their presence, and parents can access real-time, up-to-date information.

### 1.2 Core Philosophy: "Trust but Verify"
Moving away from a bottlenecked "Admin Approval" model, the platform now prioritizes **immediate action**.
- **Vendors** expect their changes to be live instantly.
- **Parents** expect the directory to be current.
- **Admins** shift from "Gatekeepers" to "Moderators", reviewing content *after* it goes live via automated alerts (Discord/Email) to catch abuse without slowing down legitimate users.

## 2. User Roles & Workflows

### 2.1 Vendors (Service Providers)
**Goal:** maximizing visibility and managing their business profile with zero friction.
- **Create Listing:** Submits business details. *Outcome:* Instantly **Live**.
- **Claim Listing:** Verifies ownership of a pre-existing listing. *Outcome:* Instantly **Verified** (upon email/token confirmation).
- **Edit Listing:** Updates pricing, photos, or description. *Outcome:* Instantly **Updated** on public site.
- **Upgrade Plan:** Moves from Free -> Standard -> Pro via Stripe. *Outcome:* Features unlock immediately.

### 2.2 Parents (End Users)
**Goal:** Finding safe, high-quality resources for their child's career.
- **Search & Filter:** Finds vendors by Category (Coach, Photographer), Region (LA, NY, SE), and Age Range.
- **Verify:** Checks for "Bonded", "Permit", and "101 Approved" badges.
- **Contact:** Directly connects via website, social, or phone.

### 2.3 Administrators
**Goal:** Ensuring platform safety and quality without blocking growth.
- **Monitoring:** Receives real-time notifications (Discord/Email) for all new/edited content.
- **Moderation:** Can retroactively "Reject" or "Hide" abusive listings.
- **Curation:** Manages the "101 Approved" badge for top-tier vendors.

## 3. Core Features & Functional Requirements

### 3.1 Listing Management (The "Engine")
The core data unit is the **Listing**.
- **Status States:**
  - `Live`: Visible to public (Default state for all new submissions/edits).
  - `Pending`: (Deprecated for general flow) Only used if automated checks fail massively.
  - `Archived/Hidden`: Removed by Vendor or Admin.
- **Data Fields:**
  - **Identity:** Business Name, Website, Contact (Email/Phone), Region (City, State, Zip).
  - **Service:** Categories (Acting Coach, Photographer, etc.), Description ("What you offer"), Unique Selling Point.
  - **Compliance:** Permit Number (for CA), Bond Number.
  - **Media:** Profile Image, Gallery Images (count based on plan).
  - **Socials:** Instagram, TikTok, YouTube, etc. (Pro only).

### 3.2 Tiered Access System
Features are gated by Subscription Plan (managed via Stripe).

| Feature | Free | Standard | Pro / Founding |
| :--- | :---: | :---: | :---: |
| **Status** | Live Instantly | Live Instantly | Live Instantly |
| **Categories** | 1 | Multiple | Multiple |
| **Gallery** | 0 Images | 6 Images | 12 Images |
| **Social Links** | No | No | Yes |
| **"Unique" Field**| No | Yes | Yes |
| **Priority** | Low | Medium | High |
| **Badge** | No | No | Eligible |

### 3.3 Image Management
- **Upload:** Direct upload to Supabase Storage.
- **Cropping:** User-defined "focal point" or crop area to ensure heads aren't cut off in card/header views.
- **Optimization:** Automatic resizing/compression for performance.

### 3.4 Search & Discovery
- **Full-Text Search:** Name, Description, Tags.
- **Facets:** 
  - **Category** (e.g., "Headshots")
  - **Region** (e.g., "Los Angeles")
  - **Age Group** (e.g., "Teens", "Kids")

## 4. Technical Architecture

### 4.1 Stack
- **Frontend:** Next.js 14+ (App Router)
- **Styling:** Tailwind CSS (Shadcn UI components)
- **Backend:** Supabase (PostgreSQL, Auth, Storage)
- **Payments:** Stripe
- **Email:** Resend / NodeMailer (Transactional)
- **Monitoring:** Discord Webhooks (Real-time logs)

### 4.2 Critical Data Flows
1.  **Submission Flow:**
    `Form Submit` -> `Server Action` -> `Zod Validation` -> `Supabase Insert/Update` -> `Revalidate Cache` -> `Discord Notify` -> **User Sees Success**.
2.  **Claim Flow:**
    `Claim Request` -> `Verify Rights (Email/User)` -> `Supabase Update (Owner ID)` -> **User Gains Control**.

### 4.3 Deduplication Strategy
- Prevent duplicates by checking:
  1.  Exact Website URL matches.
  2.  Fuzzy match on Name + City + State.
- *Action:* Block submission and prompt user to "Claim" instead if match found.
