## Listings Table

### Required Fields for Live Listings

When `status = "Live"`, these fields are **REQUIRED**:

- ✅ `listing_name` - Business/service name
- ✅ `email` - Valid email address
- ✅ `is_active` - Must be `true`
- ✅ `city` - City name (unless format is "Online")
- ✅ `state` - 2-letter state code (unless format is "Online")
- ✅ `profile_image` - Image URL (required for Standard/Pro plans, optional for Free)

### Field-by-Field Guide

#### Basic Information

**`listing_name`** (TEXT, Required)
- **Format:** Plain text, business/service name
- **Capitalization:** Use Title Case (e.g., "ABC Acting Studio", "Jane's Talent Agency")
- **Length:** Minimum 1 character
- **Examples:**
  - ✅ "ABC Acting Studio"
  - ✅ "Jane's Talent Agency"
  - ✅ "The Acting Coach"
  - ❌ "" (empty)

**`slug`** (TEXT, Auto-generated)
- **Format:** Auto-generated from `listing_name`
- **Rules:**
  - Converted to lowercase
  - Spaces replaced with hyphens
  - Special characters removed (only a-z, 0-9, hyphens allowed)
  - Multiple consecutive hyphens collapsed to single hyphen
  - Leading/trailing hyphens removed
- **Example:** "ABC Acting Studio" → "abc-acting-studio"
- **Note:** Do NOT manually set this field - it's auto-generated by database trigger

**`status`** (TEXT, Required)
- **Accepted Values:** (Case-sensitive, exact match required)
  - `"Live"` - Published and visible to public
  - `"Pending"` - Awaiting admin approval
  - `"Draft"` - Work in progress, not submitted
  - `"Archived"` - Hidden from public, kept for records
  - `"Rejected"` - Not approved for publication
- **Capitalization:** First letter uppercase, rest lowercase
- **Examples:**
  - ✅ "Live"
  - ✅ "Pending"
  - ❌ "live" (wrong case)
  - ❌ "pending" (wrong case)

**`plan`** (TEXT, Optional)
- **Accepted Values:** (Case-insensitive, but stored as provided)
  - `"Free"` - Free listing plan
  - `"Standard"` - Standard paid plan
  - `"Pro"` - Pro paid plan
  - `"founding"` - Founding member plan
- **Capitalization:** Use Title Case for consistency
- **Examples:**
  - ✅ "Free"
  - ✅ "Standard"
  - ✅ "Pro"

#### Contact Information

**`email`** (VARCHAR(255), Required for Live listings)
- **Format:** Valid email address
- **Validation:** Must match standard email format (user@domain.com)
- **Required:** Yes, when `status = "Live"`
- **Examples:**
  - ✅ "info@abcacting.com"
  - ✅ "contact@example.com"
  - ❌ "not-an-email"
  - ❌ "" (empty - not allowed for Live listings)

**`phone`** (VARCHAR(20), Optional)
- **Format:** Phone number as string
- **Length:** Maximum 20 characters
- **Format:** No strict format required, but recommended: "(123) 456-7890" or "123-456-7890"
- **Examples:**
  - ✅ "(818) 555-1234"
  - ✅ "818-555-1234"
  - ✅ "+1-818-555-1234"
  - ✅ "" (empty - allowed)

**`website`** (VARCHAR(500), Optional but recommended)
- **Format:** Full URL with protocol
- **Validation:** Must be valid URL format
- **Protocol:** Must include `http://` or `https://`
- **Examples:**
  - ✅ "https://www.abcacting.com"
  - ✅ "https://abcacting.com"
  - ✅ "http://example.com"
  - ❌ "www.abcacting.com" (missing protocol)
  - ❌ "abcacting.com" (missing protocol)
  - ✅ "" (empty - allowed, but not recommended)

#### Location Information

**`city`** (TEXT, Required for Live listings unless format is "Online")
- **Format:** City name as plain text
- **Capitalization:** Use Title Case
- **Required:** Yes, when `status = "Live"` AND `format` is NOT "Online"
- **Examples:**
  - ✅ "Los Angeles"
  - ✅ "New York"
  - ✅ "San Francisco"
  - ❌ "" (empty - not allowed for Live non-online listings)

**`state`** (TEXT, Required for Live listings unless format is "Online")
- **Format:** 2-letter US state code (uppercase recommended)
- **Required:** Yes, when `status = "Live"` AND `format` is NOT "Online"
- **Examples:**
  - ✅ "CA" (California)
  - ✅ "NY" (New York)
  - ✅ "TX" (Texas)
  - ❌ "" (empty - not allowed for Live non-online listings)

**`zip`** (INTEGER, Optional)
- **Format:** 5-digit ZIP code as integer
- **Range:** 10000-99999
- **Examples:**
  - ✅ 90210
  - ✅ 10001
  - ✅ NULL (empty - allowed)

**`region`** (TEXT[], Array, Optional)
- **Format:** Array of region names (multi-select)
- **Input Format:** Comma-separated string (will be converted to array)
- **Accepted Values:** (Case-sensitive, exact match required)
  - `"West Coast"`
  - `"Southwest"`
  - `"Southeast"`
  - `"Midwest"`
  - `"Northeast"`
  - `"Mid-Atlantic"`
  - `"Pacific Northwest"`
  - `"Rocky Mountain"`
  - `"Canada"`
  - `"Global (Online Only)"`
- **Capitalization:** Use exact capitalization as shown above
- **Multiple Values:** Separate with commas (will be converted to array)
- **Examples:**
  - ✅ `["West Coast"]` (single region)
  - ✅ `["West Coast", "Southwest"]` (multiple regions)
  - ✅ Input: `"West Coast, Southwest"` → Stored as: `["West Coast", "Southwest"]`
  - ✅ `[]` (empty array - allowed)
  - ❌ `["west coast"]` (wrong capitalization)
  - ❌ `["WestCoast"]` (wrong format)

#### Images

**`profile_image`** (TEXT, Required for Standard/Pro plans when Live)
- **Format:** Full URL to image OR filename in Supabase Storage
- **Storage:** Supabase Storage bucket `listing-images`
- **Accepted Formats:** JPEG, PNG, WebP, HEIC
- **Size Limit:** 10MB maximum
- **Recommended Size:** 1200x1200px (square, 1:1 aspect ratio)
- **URL Format:**
  - Full URL: `"https://[supabase-url]/storage/v1/object/public/listing-images/filename.jpg"`
  - Filename only: `"filename.jpg"` (will be converted to full URL)
- **Required:** Yes, when `status = "Live"` AND `plan` is NOT "Free"
- **Examples:**
  - ✅ `"https://[supabase-url]/storage/v1/object/public/listing-images/logo-1234567890.jpg"`
  - ✅ `"logo-1234567890.jpg"` (filename only)
  - ✅ `""` (empty - allowed for Free plans)
  - ❌ `""` (empty - NOT allowed for Standard/Pro Live listings)

**`gallery`** (TEXT, Optional)
- **Format:** JSON string containing array of image URLs
- **Storage:** Supabase Storage bucket `listing-images`
- **Accepted Formats:** JPEG, PNG, WebP, HEIC
- **Size Limit:** 10MB per image
- **JSON Format:** `'["url1.jpg", "url2.jpg", "url3.jpg"]'`
- **Examples:**
  - ✅ `'["image1.jpg", "image2.jpg"]'` (JSON string)
  - ✅ `""` (empty - allowed)
  - ❌ `["image1.jpg"]` (not a JSON string)

### Validation Rules Summary

**For Live Listings:**
1. ✅ `listing_name` must not be empty
2. ✅ `email` must be valid email format
3. ✅ `is_active` must be `true`
4. ✅ `city` required (unless `format = "Online"`)
5. ✅ `state` required (unless `format = "Online"`)
6. ✅ `profile_image` required (unless `plan = "Free"`)
7. ✅ If `is_bonded = true`, then `bond_number` required
8. ✅ If `is_approved_101 = true`, then `ca_permit_required` must be `true`

**For Profiles:**
1. ✅ `id` must match `auth.users.id`
2. ✅ `role` must be one of: "vendor", "parent", "admin"

## Common Mistakes to Avoid

### ❌ Wrong Capitalization
- ❌ `status = "live"` → ✅ `status = "Live"`
- ❌ `role = "Vendor"` → ✅ `role = "vendor"`
- ❌ `format = "online"` → ✅ `format = "Online"`
- ❌ `region = ["west coast"]` → ✅ `region = ["West Coast"]`
- ❌ `categories = ["acting classes"]` → ✅ `categories = ["Acting Classes & Coaches"]`

### ❌ Missing Protocol in URLs
- ❌ `website = "example.com"` → ✅ `website = "https://example.com"`
- ❌ `facebook_url = "facebook.com/page"` → ✅ `facebook_url = "https://facebook.com/page"`

### ❌ Wrong Array Format
- ❌ `categories = "Acting Classes"` (should be array or comma-separated string)
- ❌ `region = "West Coast"` (should be array or comma-separated string)
- ✅ `categories = "Acting Classes & Coaches, Audition Prep"` (comma-separated)
- ✅ `categories = ["Acting Classes & Coaches", "Audition Prep"]` (array)

### ❌ Missing Required Fields for Live Listings
- ❌ `status = "Live"` but `email = ""` → ✅ Must provide email
- ❌ `status = "Live"` but `is_active = false` → ✅ Must be `true`
- ❌ `status = "Live"`, `plan = "Standard"`, but `profile_image = ""` → ✅ Must provide image

### ❌ Invalid Category Names
- ❌ `"Acting Classes"` → ✅ `"Acting Classes & Coaches"`
- ❌ `"Headshot Photographer"` → ✅ `"Headshot Photographers"`
- ❌ `"Talent Agent"` → ✅ `"Talent Agents"`

## Notes

- **Slug Generation:** Slugs are auto-generated from `listing_name`. Do not manually set.

- **Image Storage:** Images are stored in Supabase Storage bucket `listing-images`. Provide full URL or filename.
- **Case Sensitivity:** Most text fields are case-sensitive. Use exact capitalization as shown in this guide.
- **Required Fields:** Requirements vary based on `status` and `plan`. See validation rules above.
